# Preferencias del Proyecto

## Documentación
- **NO** generar archivos de documentación (*.md) o README a menos que se solicite explícitamente
- Mantener la documentación mínima y solo cuando sea necesaria

## Estilo de Código
- Seguir las convenciones del proyecto existente
- Mantener consistencia con el código actual
- **Preferir programación funcional sobre clases**
- Usar funciones puras cuando sea posible
- Aplicar principios de **Clean Code**
- Seguir principios **SOLID**
- Aplicar **DRY** (Don't Repeat Yourself)
- Aplicar **KISS** (Keep It Simple, Stupid)
- Aplicar principios de **DDD** (Domain-Driven Design)

## Herramientas
- Usar **pnpm** como gestor de paquetes por defecto

## Comunicación
- Respuestas concisas y directas
- Explicaciones técnicas claras cuando se soliciten
